# 5-Minute Article Template (900-1,100 words)

## 1) Hook (80-120 words)
State a painful PM tension in 2-3 lines. Use one concrete situation from a growth-stage startup.

## 2) Claim + Boundary Conditions (120-160 words)
Make one strong claim. Add where it works and where it does not.

## 3) Framework Breakdown (250-320 words)
Name the framework and explain each component clearly. Keep to one primary framework (max two).

## 4) Example + Tradeoff (180-240 words)
Show a realistic decision scenario. Include at least one tradeoff and one failure mode.

## 5) Counterargument + Rebuttal (120-160 words)
Present the strongest opposing view. Rebut with logic, evidence, or explicit constraints.

## 6) Action Checklist (90-130 words)
Give 3-5 actions a PM team can execute in the next 7 days.

## 7) Closing CTA (40-70 words)
One next experiment or operating change to run this week.

## Non-Negotiables
- One idea per article.
- No generic AI hype language.
- First 150 words must be clearly relevant to growth-stage PMs.
- Include explicit startup constraints (limited data, small team, execution pressure).
